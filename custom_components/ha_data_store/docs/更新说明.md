## 1. 媒体/音响虚拟设备

### 新增设备类型

在系统配置 → 虚拟设备中新增两种设备类型：

| 类型 | 选项名 | 域 | 说明 |
|------|--------|------|------|
| `media` | 🎬 媒体虚拟设备 | `media_player` | 完整影音播放器 |
| `speaker` | 🔊 音响虚拟设备 | `media_player` | 专注音频体验 |

### 媒体虚拟设备功能

- 播放 / 暂停 / 停止 / 开关机
- 音量控制（0~1 + 静音）
- 9 种音源切换：HDMI 1-3、TV、Netflix、YouTube、电脑、游戏机、蓝牙
- 5 种音效模式：标准、影院、音乐、新闻、运动
- 上下曲切换、随机播放、循环播放、清空播放列表
- 媒体元数据显示：标题、艺术家、专辑
- 切到 Netflix/YouTube 时自动填充媒体信息

### 音响虚拟设备功能

- 播放 / 暂停 / 停止 / 开关机
- 音量控制（0~1 + 静音）
- 7 种音源切换：AUX、蓝牙、光纤、同轴、USB、WiFi、AirPlay
- 8 种音效模式：标准、流行、古典、摇滚、爵士、人声、重低音、3D环绕
- 上下曲切换

### 使用方式

```
POST /api/ha_data_store/virtual_device
{
  "device_type": "media",
  "entity_id": "media_player.my_theater",
  "device_name": "家庭影院",
  "entity_name": "客厅播放器"
}
```

重启 HA 后生效。

## 2. 属性按日分组查询 (attr_daily)

### 新查询类型

按天分组返回指定月份的属性记录，支持自动或手动指定日期字段。

### URL 格式

```
GET /api/ha_data_store/query?type=attr_daily&attr_type=ele_day&entity_id=sensor.xxx&month=2026-01&date_field=datetime
```

### 参数说明

| 参数 | 必填 | 说明 |
|------|------|------|
| `attr_type` | 是 | 属性类型名（仅单表） |
| `entity_id` | 是 | 实体 ID |
| `month` | 是 | 月份，格式 `YYYY-MM` |
| `date_field` | 否 | 日期字段名，不传则自动检测 |

### 日期字段自动检测优先级

1. 手动传入的 `date_field`
2. 表中有 `datetime` 列
3. 匹配 `on_time` / `date` / `day` / `created_at` / `updated_at`
4. 列名含 `time` 或 `date`
5. 表中第一列

### 返回格式

```json
{
  "entity_id": "sensor.xxx",
  "attr_type": "ele_day",
  "table": "attr_ele_day",
  "month": "2026-01",
  "date_field": "datetime",
  "daily_records": [
    {"date": "2026-01-31", "records": [...], "count": 24},
    {"date": "2026-01-30", "records": [...], "count": 24}
  ],
  "total_days": 31,
  "columns": ["entity_id","name","datetime",...],
  "total_rows": 744,
  "sample_date_value": "2026-01-01 00:00:00"
}
```

### 前端使用

API 工具中选择 `📅 属性按日分组`：
1. 下拉选择属性类型（单选）
2. 输入实体 ID
3. 选择月份
4. 可选：手动填写日期字段名（不填则后端自动检测）

## 3. 属性历史查询增强

### 多表支持

`attr_history` 现支持通过逗号分隔选择多张属性表同时查询。

**单表（向后兼容，返回格式不变）：**
```
GET /api/ha_data_store/query?type=attr_history&attr_type=ele_day&entity_id=sensor.xxx
```
```json
{
  "records": [...],
  "total": 123,
  "table": "attr_ele_day",
  "columns": [...]
}
```

**多表（按表名分组）：**
```
GET /api/ha_data_store/query?type=attr_history&attr_type=ele_day,ele_month&entity_id=sensor.xxx
```
```json
{
  "records": {
    "attr_ele_day": {"records": [...], "total": 123, "columns": [...]},
    "attr_ele_month": {"records": [...], "total": 12, "columns": [...]}
  },
  "total": 135,
  "tables": ["attr_ele_day", "attr_ele_month"]
}
```

### 日期范围查询

新增支持 `start` / `end` 参数进行日期范围过滤：

```
GET /api/ha_data_store/query?type=attr_history&attr_type=ele_day&entity_id=sensor.xxx&start=2026-01-01&end=2026-01-31
```

## 4. 传感器小数位数调整

### 变更说明

所有环境传感器（温度、湿度、PM2.5、CO₂、功率）的数值采集精度从 **2 位小数** 提升到 **3 位小数**。

### 修改位置

| 位置 | 原值 | 新值 |
|------|------|------|
| `_write_env_metric_record` 写入函数 | `round(value, 2)` | `round(value, 3)` |
| 采集循环 METRIC_SENSOR 类型 | `round(float(...), 2)` | `round(float(...), 3)` |

### 影响范围

- ✅ 仅对新采集的数据生效
- ✅ 历史数据不受影响（仍保留原小数位数）
- ✅ 数据库字段类型不用修改（SQLite REAL 精度足够）
- ✅ 不需要重启 HA 以外的额外操作
